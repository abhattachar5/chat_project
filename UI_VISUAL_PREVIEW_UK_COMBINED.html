Complete User Journey

1. Widget Opens (Day 1 Phase 1)

User sees FAB button, clicks to open chat widget. Consent dialog appears for voice recording.

2. Medical Document Upload (Day 2 Phase 1)

User optionally uploads medical documents. Files validated, uploaded, and scanned for conditions.

3. Review Detected Conditions (Day 2 Phase 2)

User reviews AI-extracted conditions, accepts/rejects/edits, and manually adds conditions if needed.

4. Adaptive Interview Starts (Day 2 Phase 3)

Interview begins with adaptive prompt acknowledging confirmed conditions. Prefilled answers automatically submitted.

5. Chat Interview (Day 1 Phase 2)

User answers questions via text or voice input. Messages displayed in chat tab with real-time updates.

6. Voice Input (Day 1 Phase 3)

User can use voice input with push-to-talk, see real-time transcription, and edit before submission.

7. Transcript Review (Day 1 Phase 4)

User reviews full transcript, searches, filters by role, and copies information as needed.

8. Completion

Interview completes. User sees completion screen and can download or copy transcript.
